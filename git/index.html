A story ...

One day, your boss assigns you a project. And you will lead to a team(3~4 people) to finish it.

  • QQ, Email(v1, v2, ...)
  • Setup FTP server.

Moreover

SVN, a centralized version control system.

  • Setup a SVN server or find a host, such as google code.
  • Install SVN client.

svn checkout 
svn add 
svn commit
svn log

But what if?

  • SVN server is down, or you can not connect to internet.
  • Multiple features need to develop at the same time.

Git

  • Designed and developed by Linus for Linux kernel development.
  • Distrubuted, every user has a complete copy of the repository data stored locally.
  • An active community: GitHub.

3 states in Git

// working -> index.
git add <file>
// stage -> history, move HEAD forword.
git commit

git commit

git diff

git diff

Git Branch:

  • master(Default)
  • Each feature belongs to a branch.

git branch <brach_name>

Git remote

Sync with remote repository.

git clone <url>
git remote -v
git remote add <name> <url>
git push <name> <branch>

SVN vs Git

SVN:

  • Linear
  • a single repository host on a remote server. (Company)
  • still used widely

Git:

  • DAG
  • Design for open-source workflow
  • More flexible
